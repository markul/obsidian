## Key Decisions

- Use `Skp.Kafka.Inbox 12.0.13` with the default serializer
- Use the Confluence allowlist `NB302`, `GP302`, `EI302`, `CD148`, `CR101`; the earlier temporary `TEST1` and `TEST2` note is obsolete
- Consume the DWH Kafka message as raw JSON using built-in inbox `AddOpenApiConsumer`; do not add a custom raw JSON consumer while the package helper covers this case
- Keep the transport notes aligned with the verified producer shape `AppNegatives[]` even though older docs and earlier local notes used a singular `AppNegative`
- For local Skaffold logs, Skaffold `v2.18.2` does not support `skaffold dev --container`; use `skaffold dev -p api --port-forward --tail=false` and tail only the API container with `kubectl logs`
- Do not include Redis-based inbox result publishing unless deployment requirements later require it
- Treat LM lookup failures as non-fatal for inbox processing: log the error and return success
- Keep collection argument matching in handler tests on the repo-native `.ToEquivalent()` helper; do not keep a custom `ItIs` helper here
