## Notes

**Errors observed (Windows host, Minikube cluster):**
- When executed via `bash`: `/bin/bash: line 8: /src/test/Ufr.Kpnsb.LimitService.Tests.Integration/db-init/init-db.sh: cannot execute: required file not found`
- When executed via `sh` manually: `init-db.sh: 2: set: Illegal option -`

**Root cause:** `init-db.sh` was checked out with Windows CRLF line endings. The shebang line becomes `#!/bin/sh\r` — Linux kernel cannot find interpreter `/bin/sh\r` (with trailing `\r`).

**Fix 1:** Added `.gitattributes` at repo root with `*.sh text eol=lf` — forces Git to store and checkout shell scripts with LF on all platforms.

**Fix 2:** Added `sed -i 's/\r$//'` in [test/Ufr.Kpnsb.LimitService.Tests.Integration/Dockerfile](../../../../../../dev/git/alfa/ufr-kpnsb-limit-service/test/Ufr.Kpnsb.LimitService.Tests.Integration/Dockerfile) before `chmod +x` — strips `\r` at Docker build time as a safety net regardless of checkout line endings.
