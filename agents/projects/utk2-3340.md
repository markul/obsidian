## Key Decisions

- Use `postgresql16-client` in the Alpine SDK-based test image because this service's local Skaffold stack uses PostgreSQL 16.
- Keep existing test image default behavior as `dotnet test`; integration tests are responsible for starting cron job processes from the copied publish output.
- Use Docker host networking for local image build validation when the Alfa Alpine mirror is accessed from Docker BuildKit.
- Treat Docker daemon DNS as a local environment issue until fixed; the deterministic workaround for Alfa mirror access is host networking.
- Treat `skaffold dev` port-forward startup as insufficient by itself for host-run integration tests; verify expected endpoints before `dotnet test`.
- Before running host-side integration tests, ensure no old `skaffold` or `kubectl port-forward` processes are holding the expected ports.
